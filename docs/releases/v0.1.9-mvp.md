# v0.1.9-mvp

This release brings the public package up to the current post-MVP browser shell policy,
accessibility, and visual identity work without changing the core `showModalDialog`
synchronization boundary.

## Changes

- The wrapper now supports a JSON administrator browser shell policy for hiding selected toolbar
  controls, hiding the full primary toolbar, exporting/applying policy files, and resetting only
  user-managed startup and compatibility settings.
- Targeted navigation accelerator suppression is available through the same shell policy for the
  measured Back, Forward, and Reload command group while preserving WebView2 find-in-page.
- Compatibility status is now a compact toolbar control with detail UI, and revoking permission on
  an already loaded document returns to the discovery/consent path instead of appearing stalled.
- The shell has a distinct brown Improvised EOSL visual identity, updated application icon,
  page-title window captions, and a WebView2-backed `Ctrl+F` find shortcut.
- Documentation now records browser shell policy behavior, manual validation results, navigation
  accelerator research, IE keyboard-event mutation research, visual redesign validation, and the
  current public-facing README state.

## Explicit limits

- Browser shell policy controls wrapper presentation and a measured navigation-shortcut group; it
  is not kiosk, DLP, lockdown, native-close suppression, or a navigation allow-list.
- The policy does not disable WebView2 security, Chromium sandboxing, or all browser accelerator
  keys. `Ctrl+F` and `F3` find-in-page remain intentionally available.
- Dedicated Browser Back / Forward hardware keys remain untested unless target deployment hardware
  introduces that requirement.
- IE keyboard-event mutation is documented as research only. Complete IE DOM compatibility,
  ActiveX/COM, Trident rendering, enterprise deployment tooling, signing, update support, and
  production compatibility guarantees remain outside this experimental MVP.

# v0.1.5-mvp

This experimental Windows package brings the downloadable ZIP up to date with the completed
shell and local-content issues merged after `v0.1.4-mvp`.

## Changes

- Local `.html` and `.htm` files can be opened in the existing main WebView2 by drag-and-drop,
  absolute Windows path, or `file:` URL. Content is served through a bounded loopback-only server
  so relative resources and relative `showModalDialog` URLs continue to work without enabling
  direct `file:` compatibility origins.
- The address bar now keeps a stable layout instead of resizing its navigation action with the
  URL text.
- The main window persists validated normal restore bounds and maximized state. Closing while
  minimized preserves the state that preceded minimization, and placement from a disconnected
  display is rejected safely.
- The executable and main window now carry a multi-resolution application icon.
- Back, forward, reload, navigate, compatibility settings, and diagnostics now have consistent
  browser-shell icons, tooltips, keyboard focus, and automation names.

## Validation

- solution build completed with 0 warnings and 0 errors
- all 47 policy tests passed
- the generated self-contained `win-x64` package completed all three `--auto` synchronous
  child-dialog cycles and exited with code 0
- `git diff --check` passed
- prior manual validation covers local-content loading, window placement and minimized close,
  removed-monitor fallback, title/taskbar/executable icons, command accessibility, and current-DPI
  icon rendering

## Explicit limits

- Literal Explorer-to-WebView2 drag automation remains environment-limited; address-bar loading
  and the underlying drop policy are validated separately.
- Exact placement across display-scale changes remains unmeasured.
- `window.open()` legacy feature emulation is not included; that work remains separate.
- This remains an experimental technical preview, not a production compatibility guarantee or a
  supported Internet Explorer replacement.

The ZIP includes the .NET 8 runtime. Microsoft Edge WebView2 Runtime remains required.

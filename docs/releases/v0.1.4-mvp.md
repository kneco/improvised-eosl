# v0.1.4-mvp

This experimental Windows package adds a small legacy-EC comparison workflow for the
supported synchronous modal surface.

## Changes

- Added `レガシーECサンプル` to the local home page.
- The sample uses a same-origin HTTP parent page and a separate HTTP quantity dialog.
- The parent keeps the synchronous legacy call shape and consumes the child return value.
- The child reads `window.dialogArguments`, sets `window.returnValue`, and calls `window.close()`.
- First-call discovery returns `undefined` safely instead of adding an invalid cart item.
- Unsupported `window.open` feature emulation and ActiveX are visibly separated from the
  supported `showModalDialog` path.
- Child native window titles now follow the child document `<title>`, with `Dialog` as a
  bounded fallback, instead of exposing the internal spike name.

## Download and start

1. Download `ImprovisedEosl-0.1.4-mvp-win-x64.zip` from the release assets.
2. Extract the entire ZIP to a writable folder.
3. Double-click `ImprovisedEosl.Spike.SyncModal.exe`.
4. Open `レガシーECサンプル` from the home page.
5. Add a product, allow compatibility and reload if prompted, then add it again.

## Validation

- solution build completed with 0 warnings and 0 errors
- all 42 policy tests passed before the UI comparison run
- all three `--auto` synchronous child-dialog cycles passed after the changes
- end-to-end UI verification returned quantity `3` for a 98,000-yen item and rendered a
  294,000-yen cart total
- the child window visibly used the document title `数量入力`

The ZIP includes the .NET 8 runtime. Microsoft Edge WebView2 Runtime remains required.
This remains an experimental technical preview, not a production compatibility guarantee.

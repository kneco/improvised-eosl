This release completes the selected `window.open()` compatibility work.

## Changes

- Feature-bearing `window.open()` calls can open modeless, same-origin HTTP(S) child windows after
  explicit per-origin consent.
- `scrollbars=yes/no` and `status=yes/no` are captured through a bounded synchronous bridge and
  applied to the native child host. Omitted scrollbars remain visible and usable for safety.
- Legacy compatibility consent now offers allow all currently known features, allow a selected
  subset, or deny all. Grants and denials are persisted separately and can be cleared from the
  compatibility settings window.
- The consent and compatibility-decision UI is available in Japanese.
- Edge IE-mode and WebView2 observation harnesses document the measured feature behavior and the
  intentional compatibility limits.

## Explicit limits

- `location`, `fullscreen`, and `channelmode` remain unsupported.
- `resizable`, `menubar`, and `toolbar` remain approximated or host-defined as documented in the
  implementation plan.
- Compatible child windows are restricted to explicitly permitted origins and same-origin HTTP(S)
  targets. Cross-origin popup emulation is intentionally blocked.
- This remains an experimental technical preview, not a production compatibility guarantee or a
  supported Internet Explorer replacement.

The ZIP includes the .NET 8 runtime. Microsoft Edge WebView2 Runtime remains required.

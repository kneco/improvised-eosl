# v0.1.10-mvp

This release candidate collects the post-`v0.1.9-mvp` shell usability and function-key policy work
without changing the core `showModalDialog` synchronization boundary.

## Changes

- The wrapper address entry now selects the full displayed URL when it receives focus, matching the
  measured modern-browser expectation for quick replacement while still allowing caret placement
  after the field is already focused.
- The previous Settings, Diagnostics, and compatibility status toolbar entries are consolidated:
  the toolbar keeps a single Settings gear, diagnostics move into the shell settings/help hub, and
  compatibility state is visible as an embedded address-entry chip.
- F1 now opens the shell settings/help hub instead of depending on legacy `onhelp` behavior or page
  keyboard mutation.
- Browser-like function-key actions are administrator configurable through `functionKeyPolicy`:
  F5 reload, F6 address focus, F11 wrapper fullscreen, and F12 WebView2 DevTools availability each
  default to enabled and can be explicitly disabled.
- F6 gives priority to the visible wrapper address entry. If the address entry is hidden or
  unavailable, F6 is handled as ignored instead of moving focus to hidden browser UI.
- Dedicated measurement fixtures and documentation now record the distinction between IE keyboard
  event mutation research and wrapper-owned function-key browser actions.

## Explicit limits

- Function-key policy is wrapper/browser-shell policy, not IE DOM compatibility, keyboard-event
  mutation, arbitrary page script rewriting, or per-keystroke native bridging.
- The policy does not disable WebView2 security, Chromium sandboxing, or all browser accelerator
  keys. `Ctrl+F` and `F3` find-in-page remain intentionally available.
- F11 fullscreen is a wrapper-owned fullscreen toggle, not kiosk mode, DLP, native-close
  suppression, or an enterprise lockdown guarantee.
- F12 DevTools availability is controlled through WebView2 settings and should be configured by an
  administrator policy file before loading target applications.
- Complete IE DOM compatibility, ActiveX/COM, Trident rendering, enterprise deployment tooling,
  signing, update support, and production compatibility guarantees remain outside this experimental
  MVP.

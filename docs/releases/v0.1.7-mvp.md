# v0.1.7-mvp

This experimental Windows package completes the selected broader product-MVP feature set with
portable user settings and bounded dummy-launcher `window.close()` handoff compatibility.

## Changes

- The toolbar now has one unified settings window for the next-start initial URL and user-managed
  per-origin compatibility decisions.
- User settings can be exported and imported as bounded JSON. The settings window includes an
  explicit drag-and-drop target; trusted configured profiles and WebView2 browser data remain
  outside the portable file.
- A same-origin dummy launcher can open its first business child and then request its own
  `window.close()`. The wrapper retains the original parent WebView2, applies the captured child
  name/size/chrome, and navigates it once to the business URL.
- `window.close handoff` is a separate compatibility decision. It does not grant arbitrary native
  window-close authority and is revalidated against the current origin at the native boundary.
- An `about:blank` staging child prevents the business URL from executing twice while preserving a
  usable synchronous `window.open()` return. If no parent close follows in the same task, the child
  is released and navigates as a normal popup.
- MVP release exit criteria and the remaining post-MVP issue classification are now explicit.

## Explicit limits

- The close handoff targets an immediate dummy-launcher pattern: one same-origin HTTP(S) first-child
  GET request followed by parent close in the same task.
- A real child JavaScript heap, POST body, complete `WindowProxy`/`window.opener` behavior,
  `document.write()` into `about:blank`, multiple successors, and general IE DOM compatibility are
  not preserved.
- A `window.close()` request without an eligible pending child does not close the native wrapper.
- ActiveX/COM, Trident rendering, enterprise deployment tooling, signing, update support, and
  production compatibility guarantees remain outside this MVP.

The ZIP includes the .NET 8 runtime. Microsoft Edge WebView2 Runtime remains required.

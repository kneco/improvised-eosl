# v0.1.8-mvp

This patch release makes the experimental Windows package easier to extract and start without
changing the browser compatibility or security boundaries.

## Changes

- The release ZIP no longer contains a redundant versioned wrapper directory. After extracting
  the complete archive into a new folder, `ImprovisedEosl.Spike.SyncModal.exe` is visible at that
  folder's root.
- Managed dependencies, the WebView2 native loader, `config`, and `pages` retain their tested
  positions relative to the executable. This remains a folder-based package, not a single-file
  application.
- A reusable archive-layout check requires the executable, README, project license, third-party
  notices, native loader, configuration, and HTML pages.
- Pull requests now build and validate the Windows ZIP before merge. Tag releases continue to run
  the same package checks before creating the GitHub Release.

## Validation

- all 60 policy checks passed locally
- the self-contained `win-x64` Release publish completed
- the archive-layout check passed for all 499 ZIP entries
- the extracted executable completed `--auto` with exit code `0`
- the pull-request Windows package workflow passed
- `git diff --check` passed

## Explicit limits

- Extract the complete ZIP into a new writable folder; do not run the executable inside the ZIP or
  move it away from adjacent files.
- Microsoft Edge WebView2 Runtime remains required.
- ActiveX/COM, Trident rendering, enterprise deployment tooling, signing, update support, and
  production compatibility guarantees remain outside this experimental MVP.

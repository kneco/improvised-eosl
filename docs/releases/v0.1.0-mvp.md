# v0.1.0-mvp

## Status

This tag marks the technically complete experimental MVP for synchronous
`window.showModalDialog` compatibility.

It is intended for isolated demonstrations and technical review. It is not a production
release, a supported Internet Explorer replacement, or an enterprise deployment package.

## Included

- synchronous parent JavaScript blocking without rewriting the legacy call site to async
- interactive child WebView2 on a dedicated STA thread
- JSON-compatible `window.dialogArguments` and `window.returnValue`
- cancellation as JavaScript `undefined`
- shared cookies and `localStorage` through a common WebView2 user data folder
- measured MVP feature-string behavior for size, position, center, and resize
- application-modal native HWND ownership, including bounded nested dialogs
- ordinary-site browsing with per-origin legacy API discovery, consent, persistence, and revocation
- configured compatibility profiles and startup URL selection
- bounded payloads, navigation restrictions, origin verification, and rolling diagnostics
- child renderer, shared browser-process, and persistent parent-renderer recovery paths

## Validation

- .NET SDK `8.0.422`
- WebView2 Runtime `149.0.4022.98`
- OS `Microsoft Windows NT 10.0.26200.0`
- 42 pure policy tests passed
- all 15 WebView2 automatic modes passed serially
- manual child interaction, resize, title-bar X, modality, and ordinary-site browsing passed
- Edge IE mode reference measurements are recorded for dialog arguments and feature behavior

The primary validation display was `1920x1080` at WPF DPI scale `1x1`. A secondary
`1536x864` logical display was connected to the left, but cross-display movement was not part
of the acceptance run.

## Explicit limits

- `status` and `scroll` are parsed and logged but not visually emulated.
- JSON projection does not preserve COM objects, functions, DOM objects, prototypes, cycles,
  or JavaScript object identity.
- `sessionStorage` is not shared between separate top-level WebViews.
- Renderer/browser recovery loses in-memory DOM and JavaScript state.
- Multiple wrapper processes sharing one UDF have not been recovery-tested.
- Mixed-DPI, cross-monitor, minimize/restore, taskbar grouping, packaging, signing, updates,
  deployment, and support policy remain post-MVP work.
- ActiveX, NPAPI, Trident layout, old TLS/ciphers, complete IE DOM behavior, and production
  support are outside this MVP.

# Security policy

## Supported versions

Improvised EOSL is an experimental MVP and currently has no production-supported release line or
security service-level agreement.

## Reporting a vulnerability

Do not include exploit details, credentials, private URLs, or sensitive logs in a public issue.
Use GitHub's private vulnerability reporting for this repository when available. If that channel
is unavailable, contact the maintainer through the contact method published on the `kneco` GitHub
profile and ask for a private reporting channel.

Please include the affected version, reproduction conditions, expected impact, and whether the
issue crosses an origin, host-object, local-file, or native-process boundary. Diagnostic logs may
contain URL paths and other sensitive test data; redact them before sharing.

Public compatibility bugs that do not expose a security boundary may be filed as normal issues.
